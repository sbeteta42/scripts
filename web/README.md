```bash

```

Selon le contexte, le script peut être utilisé :
- en ligne de commande ;

## Bonnes pratiques

- ne pas stocker de mots de passe en clair ;
- séparer la configuration du code quand c’est possible ;
- commenter le rôle de chaque script ;
- garder les scripts simples, lisibles et testables ;
- documenter les dépendances si le script n’est pas autonome.
